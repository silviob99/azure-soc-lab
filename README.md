# Azure SOC Lab

## Objective

This project demonstrates a secure Azure-based SOC environment using Microsoft Defender XDR and Entra ID.  
The lab simulates real-world security monitoring, detection, and incident response workflows.

## Simulated Scenario

A suspicious PowerShell execution triggered a Defender alert.  
The alert was investigated using KQL queries and endpoint telemetry.

---

## Detection Process

- Alert identified in Defender XDR
- Event logs reviewed
- KQL query used to validate suspicious activity
- Timeline analysis performed

---

## Incident Response Steps

1. Identification  
2. Containment  
3. Investigation  
4. Remediation  

---

## Lessons Learned

This lab demonstrates the importance of log visibility, endpoint monitoring, and structured incident response workflows.
